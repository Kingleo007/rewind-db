RewindDB Guide
What is RewindDB?
auto_awesomeThe Core Concept
RewindDB implements event sourcing: no mutable state is ever stored. Every state is computed by replaying an immutable event log using fold(events, S₀). This guarantees deterministic recovery from any failure.

Pages
dashboardDashboard
Live overview of all accounts, balances, event counts, and system health. Recent Events shows the latest events with color-coded dots.
historyTimeline
Full ordered event history for any stream. Click any event card to expand it and see the raw JSON payload. Filter pills let you focus on specific event types.
play_circleReplay Engine
Reconstruct state in two modes: Full Replay (every event from zero) or Snapshot-Assisted (delta only). Both produce identical validated state.
biotechFailure Lab
Inject real failures: crash, duplicate events, out-of-order, missing events, memory corruption, concurrent writes. Every test recovers deterministically.
fact_checkValidation
Mathematical invariant checks: balance conservation, no negatives, event sequence integrity, cross-account transfer consistency.
query_statsMetrics
Animated SVG benchmark comparing full vs snapshot replay. Shows the performance advantage of snapshot-assisted mode with hover tooltips.
